# CODING STANDARDS

## Naming

- Variables: descriptive nouns → user_age
- Functions: verbs → calculate_total_price
- Booleans: is_, has_, can_

---

## Functions

- Small and focused
- Single responsibility
- Prefer early returns

---

## Structure

- Avoid deep nesting (>2 levels)
- Keep logic flat and readable

---

## Code Style

- Follow PEP 8
- Explicit is better than implicit
- Prefer readability over cleverness

---

## Comments

- Explain WHY, not WHAT
- Avoid redundant comments

---

## Errors

- Use explicit exceptions
- Do not silently ignore failures
